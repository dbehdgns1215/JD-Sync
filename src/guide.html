최근 동기화 기록

    이용 조건 및 개인정보 처리방침 동의

    JD-Sync를 사용하려면 아래 내용을 확인하고 동의해야 합니다.

    1. 서비스 목적

    JD-Sync는 채용 사이트에서 사용자가 직접 즐겨찾기한 공고의 일부 일정 정보를 사용자가 지정한 Notion 데이터베이스로 동기화하는 Chrome 확장 프로그램입니다.

    2. 허용되는 사용

    JD-Sync는 개인적인 채용 일정 관리, 취업 준비 기록, 사용자가 직접 관리하는 Notion 데이터베이스 및 Notion Calendar 표시 목적으로만 사용할 수 있습니다.

    3. 금지되는 사용

    사용자는 JD-Sync를 채용 공고 데이터의 대량 수집, 상업적 이용, 재배포, 공개 공유, 제3자 제공, 각 채용 사이트의 이용약관 또는 기술적 접근 제한을 우회하거나 위반하는 목적으로 사용할 수 없습니다.

    4. 동기화 범위

    JD-Sync는 회사명, 공고명, 시작일, 마감일, 공고 URL, 지원 URL, 공고 ID, 직무 정보 등 일정 관리에 필요한 일부 메타데이터를 동기화할 수 있습니다. 공고 원문 HTML, 이미지 파일, 첨부 파일 전체를 수집하거나 Notion에 복제하지 않습니다.

    5. 콘텐츠 권리와 원문 확인

    채용 공고의 원문, 이미지, 기업 정보 및 기타 콘텐츠에 관한 권리는 각 채용 사이트, 기업 또는 정당한 권리자에게 있습니다. JD-Sync는 공고 원문을 대체하지 않으며, 정확한 공고 내용과 지원 조건은 반드시 원문 링크에서 확인해야 합니다.

    6. 개인정보 및 데이터 처리

    JD-Sync는 사용자가 입력한 Notion 토큰, Notion Data Source ID, 데이터베이스 속성명 설정, 공고 메타데이터, 최근 동기화 기록을 처리할 수 있습니다. 설정과 동기화 기록은 Chrome 확장 저장소에 저장되며, 공고 메타데이터는 동기화가 실행될 때 사용자가 설정한 Notion API 대상으로만 전송됩니다.

    7. 데이터 공유 및 보관

    JD-Sync는 별도 백엔드 서버를 운영하지 않으며 사용자 데이터를 판매하거나 광고 목적으로 사용하지 않습니다. 데이터는 사용자가 확장 프로그램 저장소를 삭제하거나, 확장 프로그램을 제거하거나, 브라우저 설정에서 데이터를 초기화할 때까지 로컬에 보관될 수 있습니다.

    8. 사용자 책임

    사용자는 각 채용 사이트의 이용약관과 관련 법령을 준수해야 하며, Notion 토큰과 Notion 데이터베이스 접근 권한을 안전하게 관리해야 합니다. 토큰이 외부에 노출된 경우 즉시 Notion에서 토큰을 재발급하거나 폐기해야 합니다.

    9. 변경

    JD-Sync의 기능, 이용 조건 또는 개인정보 처리 방식이 변경되는 경우 동의 절차가 다시 표시될 수 있습니다.
