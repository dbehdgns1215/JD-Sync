# 개인정보 처리방침

[English](./PRIVACY.en.md)

시행일: 2026-06-16

JD-Sync는 자소설닷컴에서 즐겨찾기한 채용 공고를 사용자가 설정한 Notion 데이터베이스로 동기화하는 Chrome 확장 프로그램입니다.

## 처리하는 데이터

JD-Sync는 기능 제공을 위해 아래 데이터를 처리할 수 있습니다.

- 사용자가 입력한 Notion internal integration token
- Notion parent ID, Notion API 버전, 데이터베이스 속성명 설정
- 회사명, 공고명, 시작일, 마감일, 공고 URL, 지원 URL, 자소설 공고 ID, 직무 등 자소설 공고 메타데이터
- 최근 동기화 결과, 이미 동기화한 자소설 공고 ID 등 로컬 동기화 기록

JD-Sync는 채용 공고 원문 HTML, 이미지 파일, 첨부 파일 전체를 수집하거나 Notion에 복제하지 않습니다. 동기화 대상은 일정 관리에 필요한 일부 메타데이터로 제한됩니다.

JD-Sync는 이름, 이메일 주소, 결제 정보, 건강 정보, 개인 메시지를 의도적으로 수집하지 않습니다.

## 데이터 사용 방식

JD-Sync는 공고 일정을 Notion으로 동기화하는 단일 목적을 위해서만 데이터를 사용합니다.

- 설정과 동기화 기록은 Chrome 확장 저장소에 저장됩니다.
- Notion 토큰은 Notion API 요청을 인증할 때만 사용됩니다.
- 공고 메타데이터는 동기화가 실행될 때 사용자가 설정한 Notion API 대상으로만 전송됩니다.

## 데이터 공유

JD-Sync는 별도 백엔드 서버를 운영하지 않습니다. 확장 프로그램은 아래 대상과만 통신합니다.

- Notion API (`https://api.notion.com`): 사용자가 설정한 Notion 데이터베이스에 페이지를 생성하기 위해 사용합니다.
- 자소설닷컴 (`https://jasoseol.com`): 사용자의 일반 브라우징 과정에서 공고 즐겨찾기 동작과 공고 상세 정보를 확인하기 위해 사용합니다.

JD-Sync는 사용자 데이터를 판매하거나, 광고 목적으로 사용하거나, 데이터 브로커에게 이전하지 않습니다.

## Chrome 확장 권한 정보의 제한적 사용

JD-Sync가 브라우저 권한을 통해 접근하는 정보는 사용자에게 보이는 동기화 기능을 제공하고 개선하는 데에만 사용됩니다. 이 정보는 맞춤형 광고에 사용되지 않으며, 판매되지 않고, 사용자가 직접 진단 정보나 오류 내용을 공유한 경우를 제외하면 사람이 읽도록 제공되지 않습니다.

## 데이터 보관

설정과 동기화 기록은 사용자가 확장 프로그램 저장소를 삭제하거나, 확장 프로그램을 제거하거나, 브라우저 설정에서 데이터를 초기화할 때까지 로컬에 보관됩니다.

## 보안

JD-Sync는 Notion API와 HTTPS로 통신합니다. Notion 토큰은 비밀번호처럼 취급해야 하며, 외부에 노출된 경우 Notion에서 토큰을 재발급하거나 폐기해야 합니다.

## 변경

이 정책은 확장 프로그램이 변경되면서 업데이트될 수 있습니다. 중요한 변경은 저장소 문서와 릴리스 노트에 기록합니다.
